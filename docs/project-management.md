# Gestión del proyecto – OtakuList

## Metodología
Para organizar el desarrollo de OtakuList se ha elegido **Kanban**
porque el proyecto es individual y no tiene fechas de entrega fijas
por sprint. Kanban permite gestionar el flujo de trabajo de forma
visual y flexible, moviendo tarjetas según se avanza.

## Tablero Trello
El tablero está dividido en 5 columnas:
- **Backlog**: todas las funcionalidades planificadas
- **Todo**: tareas listas para empezar en el ciclo actual
- **In Progress**: tareas en desarrollo actualmente
- **Review**: tareas terminadas pendientes de revisión
- **Done**: tareas completadas y verificadas

## Organización de tareas
Cada tarjeta del tablero representa una funcionalidad de la app.
Dentro de cada tarjeta hay un checklist con las subtareas técnicas
necesarias para completarla.

El orden de desarrollo previsto es:
1. Configuración del proyecto
2. Búsqueda de anime/manga por nombre
3. Ver detalle de un título
4. Añadir título a la lista personal
5. Puntuar un título
6. Ver y gestionar la lista personal
7. Filtrar la lista
8. Ver estadísticas
9. Añadir notas personales
